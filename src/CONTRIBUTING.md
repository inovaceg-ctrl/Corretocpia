# Guia de Contribuição

Obrigado por considerar contribuir com o site da Doces São Fidélis! 

## Como Contribuir

### Reportando Bugs
- Use a seção de Issues do GitHub
- Descreva o bug claramente
- Inclua passos para reproduzir
- Adicione screenshots se possível

### Sugerindo Melhorias
- Abra uma Issue descrevendo a sugestão
- Explique por que a melhoria seria útil
- Forneça exemplos se possível

### Pull Requests
1. Fork o repositório
2. Crie uma branch para sua feature (`git checkout -b feature/MinhaFeature`)
3. Commit suas mudanças (`git commit -m 'Adiciona MinhaFeature'`)
4. Push para a branch (`git push origin feature/MinhaFeature`)
5. Abra um Pull Request

## Padrões de Código

- Use TypeScript para type safety
- Siga as convenções de nomenclatura do React
- Mantenha componentes pequenos e reutilizáveis
- Escreva código limpo e comentado quando necessário
- Use Tailwind CSS para estilização
- Mantenha a identidade visual da marca

## Estrutura de Commits

Use mensagens de commit claras e descritivas:
- `feat: adiciona nova funcionalidade`
- `fix: corrige bug específico`
- `style: ajustes de estilo/CSS`
- `refactor: refatoração de código`
- `docs: atualização de documentação`

## Testando

Antes de submeter um PR:
- Teste em diferentes navegadores
- Verifique a responsividade mobile
- Certifique-se de que não há erros no console
- Teste todas as funcionalidades afetadas

## Dúvidas?

Entre em contato através das Issues do GitHub.
